Linux用户管理命令

最近搭建了一个用于linux服务器，涉及到服务器上用户的管理，特此记录一些用户和用户组管理的基本命令。Linux系统中文件都是以用户和用户组的形式管理。一个文件属于一个用于，也属于一个特定的组。在设置文件的读/写/执行三种权限时，都会针对用户，用户组，其他人，这三个类别设置不同的权限。

Linux权限控制方式简介

在介绍Linux用户管理和用户组管理之前，首先简单介绍下Linux文件的权限 表示方式。因为管理用户和用户组最终落脚点就是为了把不同的文件赋予不同的人和组以不同的读写权限。为了清晰的看到问题的本质。以一个常见的例子开始。在利用ll命令时，会显示出文件或者文件夹的权限，样子大概是这样：
-rwxr-xr-x 1 root wheel 430540 Dec 23 18:27 /bin/bash

其中root表示该文件属于root用户，wheel表示文件所在的组。
而-rwxr-xr-x则表示文件的具体权限，可以看到这串权限字符总共10个。linux把他分为四组，1+3+3+3=10的划分法。分别为:

-:一位，表示文件类型，横杠表示普通文件，还有例如d表示是一个目录。

rwx:三位，表示用户root的权限，有read，write，execute三个权限

r-x:三位，表示用户组wheel的权限，该组有read和execute的权限，-表示空

r-x:三位，表示所有其他用户的权限，同上。

ps：另外一种简单的数字权限表示方法，用4=r，2=w，1=x。那么rwx可以表示为4+2+1=7，可以很方便的计算上面例子的权限为755。即表示只有创建者才能读写执行，组用户和其他用户只能读和执行不能修改。在chmod命令中会经常直接用数字表示权限。例如chmod 666表示修改文件权限使得所有用户都能读写文件但是不能去执行文件。

用户和用户组管理

  1. 新建用户组gitgroup

    $ groupadd  gitgroup

  2. 新建用户test并加入组gitgroup

    $ adduser --ingroup gitgroup test

    新建过程中按照提示输入密码和个人信息。

  3. 将用户添加到一个组

    $ usermod -a -G sudo test

  4. 修改用户的默认组

一个用户可以属于多个组，并且有一个默认组，用户新建文件的时候，该文件的组属性时属于默认组。可以修改/etc/passwd文件来修改默认组。
username:x:x1003:1003 第二个1003为默认组
