最近开始使用hexo+vim+markdown，特此记录Markdown最基本语法。有了Markdown，写出漂亮的文档将显得轻而易举。文末也会推荐几款强大的Markdown的IDE环境，虽然使用它们可以不用你去关心Markdown语法。然而事实上Markdown的语法确实是非常简单，完全可以在vim上就写出漂亮的文章出来。

简介

Markdown是一种轻量级的标记语言，他的优点真的很多，也真的很轻很轻。千万不要被语法，语言这类词汇吓唬到。他的语法实质上非常简单，即使完全没有任何编程经验，也没关系。实际上也是，现在很多自由攥稿人和写作爱好者都用Markdown。在Markdown中常用的标记符号估计不会超过10个。比起HTML的标记语言少太多。学习成本也是非常低的，毫不夸张的说，花20分钟，你完全可以掌握，本文主要介绍了Markdown中最基本也是最常用的一些语法。

语法

标题

格式：若干个#+空格+标题名字，例如：

一级标题:# 一级标题
二级标题：## 二级标题
三级标题: ### 三级标题
以此类推..

文字标记

使用* 包裹文字表示斜体，使用** 包裹文字表示粗体,使用`代码`表示行内代码,使用~~ 包括文字表示删除线。

例如：斜体,粗体,code,删除线

链接

格式：使用[描述](链接地址)来为文字添加链接
实例：去往我的首页的连接为我的blog

无序列表

使用 - 表示无序列表

实例：

  • 无序列表
  • 无序列表
  • 无序列表

有序列表

使用数字和点号表示有序列表，有个小技巧是你可以全部使用 1+点号，Markdown会自动累加序号。

实例：

  1. 有序列表一
  2. 有序列表二
  3. 有序列表三

代码块

使用四个空格缩进表示代码块，例如：

int main()
{
  printf("hello word");
  return 0;
}

在段落中填写[TOC] 以显示全文内容的目录结构

标签分类

一般在文档头部使用 tags:android java来表示标签，二用category: android 来表示文章分类。

  1. 非代码，不高亮 ``` code ```
  1. 代码高亮 ```c my code … ```
