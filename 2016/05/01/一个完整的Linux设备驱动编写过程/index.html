首先，我们得了解Linux设备驱动的工作方式。Linux设备驱动的工作方式一直是Linux系统设计的一大亮点。Linux系统中将每一个驱动都映射成一个文件。这些文件就设备文件。保存在系统/dev目录。驱动程序的编写实质上就是对这些设备文件的读写操作。

编写Linux驱动程序的步骤

  1. 建立Linux驱动的骨架，主要完成装载和卸载驱动。
  2. 指定驱动的相关信息，作者、开原协议，描述等等。
  3. 注册和注销设备文件
  4. 指定设备文件读写的回调函数
  5. 编写业务逻辑
  6. 编写Makefile文件
  7. 编译Linux驱动程序
  8. 安装、卸载以及测试Linux驱动

本文将以一个最典型并且完整的例子介绍如何按照上面的这个步骤编写一个Linux驱动程序。该例子并没有访问具体硬件，而是利用设备文件作为介质与应用程序交互完成特定功能。应用程序向设备文件写入一个字符串，然后从设备文件读出来的就是该字符串中单词的个数。

系统环境Ubuntu14.04 64bit

准备工作

编译骨架程序

采用如下命令编译：
make -C /usr/src/linux-headers-3.19.0-25-generic/ M=/home/van/Desktop/test_driver/word_count
其中：

  1. -C制定了Linux内核的头文件，后面的参数是本机内核。我的Ubuntu14.04 64bit的内核linux-headers-3.19.0-25-generic。
  2. M=后面的参数为驱动程序源文件的目录。

编译成功后再驱动目录会生成好几个文件，最重要的就是驱动模块文件word_count.ko。可以自己用ls命令查看下，还另外生成了好多个文件。

安装驱动和卸载驱动

如下命令安装驱动：
# insmod word_count.ko
查看是否安装成功：
# lsmod | grep word_count
卸载驱动：
# rmmod word_count
注意，每次重新编译了驱动，重新安装时，必须先卸载。
查看Linux驱动输出的日志信息
# dmesg | grep word_count | tail -n 2
执行该命令后，我们可以看到上面填写的打印日志信息。

指定驱动相关信息

执行驱动相关信息不是必要的，但是一个完整的驱动程序都会指定这些信息。主要的信息包括

  • 模块作者：使用MODULE_AUTHOR宏指定；
  • 模块描述：使用MODULE_DESCRIPTION宏指定；
  • 模块别名：使用MODULE_ALIAS宏指定；
  • 开原协议：使用MODULE_LICENSE宏指定。

可以用如下命令查看是否已经成功创建了设备文件：

指定设备文件读写的回调函数

设备文件也是文件，而对文件的操作无非就是读和写。对于驱动程序的设备文件不同硬件读写时完成的操作是不同，因此我们需要自己指定读写函数。在上面提到的file_operations结构体中有两个函数指针成员变量read和write。我们需要实现这两个函数即可。新添加了两个函数word_count_read和word_count_write

重新编译word_count.c文件。并安装驱动。可以通过如下方法测试添加读写回调功能的驱动：
# echo 'hello elicec' > /dev/word_count
上面一句话是向设备文件中写入了一个字符串hello elicec。执行该操作会自动调用我们自己实现的文件写函数word_count_write。
接着我们从/dev/word_count中读取出数据。
# cat /dev/word_count
如果输出hello elicec则说明测试成功。

安装测试驱动

安装过程前面已经有涉及了，比较简单。这里我们主要来看看最后完整驱动的测试过程。主要有两种方法去测试。

测试方法一

在命令终端直接测试：

echo 'i love you.' > /dev/word_count
dmesg

执行完dmesg后仔细看输出信息。成功的话会显示write:word count:3。
